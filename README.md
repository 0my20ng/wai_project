# 📁 [W-AI 공모전] 원주형 AI 복지 내비게이터 프로젝트 실행 계획서

## 1. 프로젝트 개요 (Project Overview)

* **프로젝트명:** **원주 복지 내비게이터 (Wonju Welfare Navigator)**
* **핵심 목표:** 공공/민간 데이터를 통합하고 RAG(검색 증강 생성) 기술을 적용하여, 사용자의 상황(Context)에 맞는 최적의 복지 혜택을 찾아주는 AI 서비스 구현.
* **기간:** 2026.01.29 ~ (공모전 마감일)
* **팀 구성 및 역할:**
* **기획/전처리:** 데이터 수집 및 Orange 3 시각화
* **AI 개발:** Python 크롤링, 임베딩 모델 구축, 추천 알고리즘 구현
* **발표:** PPT 제작 및 스토리텔링



---

## 2. 단계별 상세 추진 계획 (Execution Roadmap)

### **STEP 1. 데이터 수집 (Data Collection Strategy)**

> **목표:** 정형 데이터(CSV/API)와 비정형 데이터(Web)를 결합한 '하이브리드 DB' 구축

**1-1. 정형 데이터 확보 (CSV)**

* **소스:** 공공데이터포털 > 한국사회보장정보원_민간복지서비스정보
* **작업 내용:**
* `pandas`로 CSV 로드 (`cp949` 인코딩 주의).
* **필수 컬럼 추출:** `사업명`, `지원대상`, `지원내용`, `신청방법`, `마감일`.
* **필터링:** 마감일이 `2099-12-31`이거나 `2026년` 이후인 데이터만 남김 (종료된 사업 제거).



**1-2. 실시간 데이터 확보 (Open API)**

* **소스:** 온라인 청년센터 > 청년정책 API
* **작업 내용:**
* Python `requests` 라이브러리로 JSON 데이터 호출.
* `pageIndex`를 반복문(Loop)으로 돌려 전체 데이터(약 3~4천 건) 수집.
* JSON 데이터를 CSV와 동일한 컬럼명(`사업명`, `지원대상` 등)으로 변환(Mapping).



**1-3. 상세 정보 크롤링 (Web Crawling)**

* **소스:** 각 정책의 상세 URL (API 결과에 포함됨) 또는 원주시청 고시공고 게시판.
* **작업 내용:**
* `BeautifulSoup` 활용, `div.content` 등 본문 태그 내 텍스트 추출.
* **목적:** 단순히 요약된 '지원 대상'이 아니라, 구체적인 자격 요건(텍스트)을 AI에게 학습시키기 위함.



---

### **STEP 2. 데이터 전처리 및 시각화 (Preprocessing - Orange 3 Focus)**

> **목표:** PPT 2번 항목(데이터 소개)에 들어갈 **'시각적 근거'** 확보

**2-1. 데이터 정제 (Data Cleaning)**

* **도구:** Orange 3 (`File` → `Select Columns` → `Preprocess`)
* **상세 작업:**
* **결측치 처리:** 지원 금액이 없는 경우 '상세 내용 참조'로 대체.
* **불용어 제거:** 공고문에 자주 등장하지만 의미 없는 단어(예: '붙임', '참조', '공고') 제거.



**2-2. 탐색적 데이터 분석 (EDA & Visualization)**

* **도구:** Orange 3 (`Word Cloud`, `Distributions`)
* **상세 작업:**
* **Word Cloud:** `지원대상` 컬럼을 분석하여 **'청년', '소득', '대학생', '원주'** 등의 키워드가 얼마나 많은지 시각화.
* *→ PPT 멘트: "분석 결과, 청년과 저소득층을 위한 정책이 60% 이상임을 확인했습니다."*


* **Bar Chart:** 지역별(원주 vs 전국), 분야별(주거, 생활비, 취업) 데이터 분포 그래프 생성.



---

### **STEP 3. AI 모델링 및 개발 (AI Development - Python Focus)**

> **목표:** PPT 3번 항목(분석 과정)에 들어갈 **'핵심 기술'** 구현

**3-1. 텍스트 임베딩 (Vector Embedding)**

* **기술:** Sentence-BERT (`sentence-transformers` 라이브러리)
* **상세 작업:**
* 한국어 성능이 좋은 모델 사용 (추천: `snunlp/KR-SBERT-V40K-klueNLI-augSTS`).
* 모든 정책의 `지원대상 + 지원내용` 텍스트를 합쳐서 벡터화(Vectorization) 수행.
* **결과물:** 정책 개수(N) x 768차원의 행렬(Matrix).



**3-2. 검색 및 매칭 알고리즘 (RAG Engine)**

* **기술:** Scikit-Learn (`Cosine Similarity`) 또는 FAISS
* **상세 작업:**
* **사용자 입력 처리:** "나 원주 사는 대학생인데 생활비가 없어" → 벡터 변환.
* **유사도 계산:** 사용자 벡터와 정책 벡터 간 코사인 유사도(0~1) 산출.
* **Ranking:** 유사도 점수가 높은 상위 3~5개 항목 추출.



**3-3. 생성형 답변 (LLM Integration)**

* **기술:** OpenAI API (GPT-4o/3.5) 또는 Google Gemini API
* **상세 작업 (Prompt Engineering):**
* *프롬프트 예시:* "너는 복지 상담사야. 아래 [검색된 정책] 내용을 바탕으로 사용자에게 친절하게 추천해주고, 신청에 필요한 서류를 1, 2, 3번으로 요약해서 알려줘."



---

### **STEP 4. 성능 평가 및 검증 (Evaluation)**

> **목표:** PPT 4번 항목(분석 결과)에 들어갈 **'객관적 수치'** 확보

**4-1. 테스트 시나리오 구축**

* 가상의 사용자 페르소나 5명 설정.
1. 원주 거주 대학생 (생활비 부족)
2. 미취업 청년 (취업 준비)
3. 한부모 가정 (양육비 필요)
4. ...



**4-2. 정량적 평가 (Metrics)**

* **Top-3 Accuracy (정확도):**
* 질문 50개를 던졌을 때, AI가 추천한 상위 3개 안에 '실제 적합한 정책'이 포함된 비율.
* *목표치:* 90% 이상.


* **유사도 점수 비교:**
* 정답 정책의 평균 유사도 (예: 0.85) vs 오답 정책의 평균 유사도 (예: 0.12) 차이 입증.



---

### **STEP 5. 발표 자료 제작 (PPT Scenario)**

> **목표:** 작년 수상작의 논리 구조를 계승하되 기술력 강조

* **[P1-2] 서론:** 정보의 비대칭 문제 제기 (뉴스 캡처).
* **[P3-4] 데이터:** 하이브리드 수집 전략 및 Orange 3 전처리 워크플로우 이미지.
* **[P5-6] 기술:** RAG 파이프라인 도식화 (임베딩 → 벡터DB → 매칭).
* **[P7-8] 결과:** Python 코드 캡처, 유사도 점수 표, Top-3 정확도 그래프.
* **[P9] 데모:** 실제 챗봇 대화 화면 예시 ("원주시 전입 대학생 학자금" 추천 장면 등).
